# QA Suite CRUD — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Escalar el seed de datos a 100+ animales por lote y crear `tests/qa/crud.spec.ts` con 15 tests E2E que cubren edición, borrado y anulación de todas las entidades (gastos, animales, lotes, ventas).

**Architecture:** Un solo `describe.serial` con `beforeAll(cleanup + seed)` / `afterAll(cleanup + seed)`, igual que `escritura.spec.ts`. L1 escala a 120 animales, L4 a 100, se agrega L6 "Lote Nelore Stress" con 90 animales vendidos en una sola venta para estresar `useAnularVenta`. L2, L3, L5 no cambian (calculos.spec.ts depende de sus valores exactos).

**Tech Stack:** Playwright, Firebase Admin (seed), TypeScript, React 18 + Firestore.

**Contexto importante antes de empezar:**
- Los botones de acción tienen atributo `title` — usar `button[title="Editar gasto"]` etc. es más robusto que seleccionar por emoji.
- Los tests son seriales: G2 modifica el monto del gasto "Alimento febrero" a 250000. G3 corre después y ve el estado post-G2. Esto es intencional.
